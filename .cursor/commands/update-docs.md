# Update Docs

Use this command when a change affects project direction, architecture, scope or permanent workflow truth.

## Workflow

1. Identify affected documents.
2. Update `MANIFEST.md` if the change affects scope, phases, principles or direction.
3. Update relevant docs under `docs/`.
4. Update ADR if the change is decision-level.
5. Update Cursor rules or commands if the change must persist in agent behavior.
6. Verify that assumptions, risks and open questions remain explicit.

# REAPER Scaffold

Use this command when planning or implementing scaffold generation for REAPER.

## Workflow

1. Start from canonical or mapped data, not raw source parsing.
2. Define track creation, naming, role grouping, markers and regions.
3. Preserve traceability from source fragments to scaffold outputs.
4. Ensure the result is editable and review-friendly.
5. Emit warnings or unresolved markers where the scaffold is incomplete or uncertain.

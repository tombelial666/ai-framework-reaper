# REAPER Scaffold

## Use This Skill For

- track planning;
- scaffold artifact planning;
- marker and region strategy;
- review-friendly REAPER output design.

## Default Workflow

1. Start from canonical or mapped data, never from raw source parsing.
2. Define track roles, names and ordering.
3. Define marker and region behavior.
4. Preserve traceability to source fragments.
5. Surface unresolved areas for manual refinement.

## Boundary Rules

- Do not own domain logic in REAPER integration.
- Do not present scaffold output as final arrangement truth.
- Prefer editable structure over clever but opaque automation.
